Ayuda: Plantillas

Descripción

XOOPS permite cierta separación entre la presentación real del sitio (la "skin" o interfaz de usuario) y el código que se ejecuta en segundo plano. La primera se gestiona mediante un sofisticado mecanismo basado en temas gráficos. Un tema puede incluir, en su forma básica, las plantillas HTML esenciales que definen la interfaz visual del sitio, así como las hojas de estilo e imágenes necesarias para complementar esa interfaz visual. Los temas se pueden cambiar en la interfaz administrativa, por lo que cambiar toda la apariencia de un sitio XOOPS es tan simple como cambiar el valor de un selector y hacer clic en "Aceptar". XOOPS también proporciona a los administradores del sitio la capacidad de permitir a los usuarios elegir entre algunos temas disponibles.

En términos generales, un tema es el marco visual de alto nivel de un sitio: el tema define, por ejemplo, el diseño general del sitio; por otro lado, generalmente no define la distribución específica del contenido dentro de un bloque. Esto se gestiona mediante una entidad diferente llamada conjunto de plantillas. Un conjunto de plantillas es un conjunto de plantillas HTML que definen la presentación del contenido dentro de todos los bloques que forman un sitio web XOOPS. Si queremos una metáfora más clara para esto, podríamos decir que el tema es como el arquitecto de una casa, mientras que el conjunto de plantillas es como el decorador de interiores. El primero coloca las paredes y define la distribución general, pero es el segundo quien se encarga de "vestir" esas paredes.

Si el tema activo es uno simple que solo incluye un archivo de tema, archivos CSS y algunas imágenes, XOOPS emplea el conjunto de plantillas predeterminado para definir la presentación de los bloques. Sin embargo, algunos temas complejos son lo suficientemente sofisticados como para incluir su propio conjunto de plantillas. En este caso, el administrador del sitio debe utilizar la interfaz administrativa para configurar tanto el tema como el conjunto de plantillas.

¿Cómo se logra la magia del tema? Es decir, ¿cómo pueden las plantillas HTML incluir contenido dinámico? Esto no se hace con espejos y humo, sino con un motor de plantillas inteligente llamado Smarty que realiza la función de lo que solía llamarse interpolación de variables. Expresado de manera simple, podríamos decir que el sistema XOOPS utiliza archivos PHP para mantener la lógica de los módulos (acceso a la base de datos, asignación de variables, gestión de derechos...), archivos HTML con variables Smarty insertadas para gestionar la interfaz visual real, y el motor Smarty para insertar valores y cadenas en lugares específicos dentro del HTML que construye las páginas vistas por el usuario final. En resumen, lo que esto significa para usted se puede resumir fácilmente: gracias a su sistema de temas, XOOPS permite a los diseñadores de sitios construir sitios más interesantes y visualmente impresionantes sin mucha preocupación por lo que sucede bajo el capó.

Clonación y descarga de conjuntos de plantillas

Si desea crear un conjunto de plantillas personalizado, puede hacer una copia del conjunto predeterminado (o cualquier otro) seleccionando el enlace 'clonar' (se le pedirá que le dé un nombre al nuevo conjunto). A diferencia del predeterminado, otros conjuntos de plantillas se pueden editar libremente. También puede guardar una copia de un conjunto de plantillas en el disco seleccionando el enlace 'descargar', o eliminarlo con el enlace 'eliminar'. También puede ver a veces un enlace adicional 'Generar' junto a los nuevos módulos para los que aún no se han creado plantillas.

Edición de plantillas

Puede ver una lista completa de las plantillas disponibles navegando por la vista de árbol del directorio. Cuando selecciona una plantilla, XOOPS la abrirá en un editor donde puede editarla.

Al editar plantillas, debe habilitar una preferencia que actualice automáticamente los archivos .tpl de la plantilla del módulo si hay archivos más nuevos en el directorio themes/su tema/templates para el tema actual (Módulo del Sistema -> Preferencias -> Configuración General -> ¿Actualizar los archivos .tpl de la plantilla del módulo del directorio themes/su tema/templates? -> 'Sí'). Esto debe desactivarse una vez que el sitio se haga público.

Si accidentalmente estropea una plantilla, no olvide que puede recuperar una copia 'buena' del conjunto de plantillas predeterminado. Si está trabajando con plantillas no estándar, puede ser una buena idea guardar una copia en el disco antes de comenzar a modificarlas.
